# American Barber Institute — Website

Marketing & lead-generation website for the **American Barber Institute (ABI)** — New York's dedicated barber school (Manhattan + Bronx). Zero-framework static site (HTML / CSS / vanilla JS) generated by small Python build scripts and deployed to Vercel.

**Live:** https://abi-website-black.vercel.app

---

## Tech stack

- **Output:** static HTML/CSS/JS — no runtime framework, fast and cheap to host.
- **Build:** Python 3 (standard library only).
  - `build.py` — base template + content partials (`src/pages/*.html`) → main site pages, sitemap, robots.
  - `build_landing_pages.py` — bilingual splash/landing pages (EN/ES) + the public `/` home.
  - `src/make_gallery.py`, `src/make_blog.py` — generate the gallery and blog partials.
  - `src/deploy_vercel.py` — deploy to Vercel via REST API (no CLI/Node needed).
- **Hosting:** Vercel (`cleanUrls: true`, `trailingSlash: false`).

## Build & preview locally

```bash
python3 build.py                 # build main site pages
python3 build_landing_pages.py   # build splash / EN-ES landing pages
python3 -m http.server 8000      # preview at http://localhost:8000
```

## Deploy

```bash
VERCEL_TOKEN=xxxxx python3 src/deploy_vercel.py     # production deploy
```
GitHub pushes to `main` also trigger Vercel auto-deploys.

## Design system

- **Identity:** premium barber academy — black / off-white with blue (default) + gold/red accents. Switchable themes (blue, midnight, classic, emerald, noir) via the top-bar dots.
- **Logo:** animated barber-pole GIF (`assets/img/abi-logo.gif`) in header + footer; clickable to home.
- **Calling model:** two fixed pills in the sticky header — **English (212) 290-2289** and **Español (212) 290-0278**; mobile "Call Now" opens the same two options. No third call button.
- **Language:** one-click **EN/ES** toggle beside the logo.
- **Per-page backgrounds:** each page uses a distinct, client-approved photo as a frosted-glass background (visible behind content, still on scroll, responsive). Image paths are root-absolute (`/assets/img/...`) so they resolve on every route.
- **Type:** Poppins/Oswald/Inter; medium, consistent scale; no sub-12px text on mobile.
- **Accessibility:** visible focus states, 44px+ tap targets, `prefers-reduced-motion` support, semantic headings, alt text.

## SEO / AEO

TradeSchool / LocalBusiness + AggregateRating, Course, Person, BreadcrumbList and FAQPage JSON-LD; OpenGraph + Twitter cards; `hreflang` EN/ES; `sitemap.xml` with `lastmod`; `robots.txt`; `llms.txt` for answer engines. Real next-class date and trust numbers are baked into the HTML (not JS-only) so crawlers and no-JS visitors see real values.

## Repo layout

```
build.py, build_landing_pages.py   # generators
src/pages/*.html                   # content partials (edit these, not the built files)
src/*.py                           # gallery / blog / deploy helpers
assets/css/                        # style.css, brand.css, landing.css (main), effects.css
assets/js/                         # main.js, landing.js, effects.js
assets/img/                        # images (logo + curated photos)
*.html, programs/, es/, blog/      # BUILT output (regenerated by the scripts)
```

> Edit content in `src/pages/`, styles in `assets/css/landing.css`, then re-run the build scripts. Do not hand-edit the generated top-level `*.html`.
